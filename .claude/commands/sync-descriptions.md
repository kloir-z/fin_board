# sync-descriptions

`src/lib/descriptions.ts` に登録されていない銘柄をDBから検出し、日本語説明文を追記するコマンドです。

## 手順

### Step 1 — 未登録シンボルを検出

```bash
node .claude/scripts/check_descriptions.cjs
```

出力が `✓ 全シンボルに説明文が登録されています。` なら作業不要。終了してください。

未登録シンボルが表示された場合は Step 2 へ。

### Step 2 — 説明文を調査・作成

未登録シンボルそれぞれについて、以下の情報をもとに **日本語1〜2文** の説明文を作成する：

- シンボル名・銘柄名（スクリプト出力から取得）
- 所属ウォッチリスト名（業種・地域の手がかりになる）
- あなた自身の知識（企業の事業内容・特徴・業界での立ち位置）

**説明文の基準：**
- 1〜2文、簡潔に
- 投資家が「この会社は何をしているか」を即座に理解できる内容
- 企業名・ブランド名・代表的な製品/サービスを含める
- 専門用語はなるべく噛み砕く
- 日本株は日本語社名を優先して記載

### Step 3 — descriptions.ts に追記

`src/lib/descriptions.ts` の末尾の `}` の直前に、既存の書式に合わせて追記する：

```typescript
  // ── カテゴリ名（追加日など任意コメント）──
  SYMBOL:
    '説明文（1〜2文）。',
```

**書式の注意：**
- キーはシングルクォートで囲む（`^`・`.`・`=`・`-` を含む場合は必須）
- 末尾のカンマを忘れずに
- 既存のカテゴリコメントがあればそこに追加、なければ新規コメントブロックを作る

### Step 4 — 再突合で確認

```bash
node .claude/scripts/check_descriptions.cjs
```

`✓ 全シンボルに説明文が登録されています。` が表示されたら完了。

### Step 5 — コミット

```
git add src/lib/descriptions.ts
git commit -m "feat: 説明文を追加（新規銘柄 X件）"
```

## 注意事項

- 同じシンボルが複数のウォッチリストに登録されている場合、説明文は1つで共用される
- 銘柄名がウォッチリストによって異なる場合（例: TSLA が「Tesla」と「Tesla（Optimus）」）でも、symbol が同じなら説明は1つ
- DB が起動していなくてもスクリプトは動作する（SQLiteファイルに直接アクセス）
- `data/fin_board.db` が存在しない場合はエラーになる（初回起動前は使用不可）
